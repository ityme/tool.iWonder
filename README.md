任务：创建 VS Code 插件 - 智能输入法切换器
目标：
为 Visual Studio Code 开发一个扩展插件，该插件能够在不同编辑区域智能地管理和切换输入法，以优化编码体验。

运行环境：

操作系统： Windows 11
目标软件： Visual Studio Code
本地输入法配置：
系统默认输入法：美式键盘（英文）
辅助输入法：任意中文输入法（例如：微信输入法、微软拼音等）
输入法切换快捷键：Shift
插件核心功能需求：

默认状态（代码编辑区）：

当光标在代码编辑区域（非注释、非字符串）时，强制锁定为英文输入法。
在此状态下，禁用 Shift 键切换输入法的功能，确保编码时始终为英文输入。
特定区域（允许中文输入）：

当光标进入以下区域时，插件应允许用户通过 Shift 键自由切换中/英文输入法：
代码注释区 (例如 //... 或 /*...*/ 内部)
字符串常量区 (例如 "..." 或 '...' 内部)
VS Code 侧边栏 (如文件浏览器、搜索栏等)
VS Code 集成终端
当光标离开这些区域并返回代码编辑区时，输入法应自动切换回英文，并再次锁定。
触发机制：

所有输入法状态的判断和切换逻辑，都应由 光标位置的变动 ( cursor position change ) 或 活动编辑器/窗口的焦点变化 ( active editor change ) 来触发。
技术实现参考：

请分析并参考现有插件 auto-shift-input 的实现逻辑。
项目地址：https://github.com/3biubiu/auto-shift-input
重点分析：
它是如何检测当前光标所在位置的上下文（如注释、字符串、普通代码）的？
它是如何与操作系统（Windows）交互以查询和切换当前输入法的？
它是如何监听 VS Code 的事件（如光标移动、窗口切换）的？
最终交付：

提供该 VS Code 插件的完整源代码。
代码中需要有清晰的注释，解释关键部分的逻辑。
